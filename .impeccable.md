# Chess Analyzer

## Design Context

### Users
Chess.com players who want free, move-by-move game analysis without a paid
subscription. They arrive curious about a specific game — "where did I go
wrong, what was brilliant" — and read the review at a calm pace, often on a
laptop. The job: understand a game deeply, the way an annotated games book
explains it.

### Brand Personality
Studious, literary, trustworthy. Three words: **annotated, refined, classic**.
The interface should feel like a well-set chess column in a newspaper or a
hardcover tournament book — authoritative but warm, never gamer-flashy.

### Aesthetic Direction
Light "editorial" theme. Warm cream paper background, ink-dark text, a single
crimson accent. Classic chess-book feel: strong typography hierarchy, generous
margins, thin rules, subtle paper-like shadows. Move annotations read as prose.
- Anti-references: neon-on-dark "esport" UIs, glassmorphism, gradient text,
  card-in-card dashboards, generic rounded-shadow boxes.

### Design Principles
1. **Read like a book.** Prose explanations in a literary serif; data and
   notation in a clean grotesque with tabular figures.
2. **One accent.** Crimson is reserved for primary action and emphasis; move
   classifications keep their own semantic colors (teal/blue/green/…).
3. **Paper, not panels.** Favor rules, whitespace, and tints over heavy boxes;
   never nest cards.
4. **Calm motion.** A single orchestrated reveal on analysis load; no bouncy
   micro-animations.
5. **Tabular truth.** Evals, win%, ratings, counts always align on tabular nums.
